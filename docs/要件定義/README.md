# 要件定義

[要求](../要求定義/README.md) を満たすために、システムが「何をすべきか」を記述するレイヤ。

## 書き方

- 各要件は `REQ{番号}-{テーマ}/README.md` で定義し、より細かい粒度は `REQ{番号}.{サブ番号}-*.md` に分ける
- 対応する[要求](../要求定義/README.md)へのリンクを本文の冒頭に張る（親フォルダの README だけでなく、子ファイル `REQ*.{サブ番号}-*.md` にも張ること）
- 下位の[仕様](../仕様/README.md)へのリンクは張らない（リンクグラフが双方向に濁るため、参照は仕様側から上向きに張る）
