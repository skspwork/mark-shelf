# REQ6.1: SSE による変更通知

[R1.4 — 保存即反映](../../要求定義/R1-ドキュメント閲覧性/R1.4-保存即反映.md) を満たすため、サーバー側でファイル変更を監視し、Server-Sent Events でクライアントに通知する。

## 要件

- ドキュメントルート配下の変更（作成・更新・削除・リネーム）を検知する
- 変更検知時にクライアントへイベントを送信し、ツリーと本文が自動で再取得される
- `node_modules`・`.git`・`.next`・`dist`・`.turbo` など明らかに無関係なディレクトリは監視対象外
- 接続が長時間続いてもプロキシやブラウザに切断されないよう keep-alive を送信する
